# Agent Context: SZ Calculator

This document provides essential context for any AI agent or developer picking up this project. It summarizes the architecture, design decisions, and improvements made over time.

## Project Overview

A high-fidelity, full-stack calculator application inspired by the iOS design language.

- **Frontend**: React (TypeScript) + Vite, structured with a modular, hook-based architecture.
- **Backend**: Go microservice using the Huma REST framework.
- **Infrastructure**: Dockerized with separate Development (hot-reload) and Production configurations.

## Critical Architectural Rules

1. **Zero Frontend Math**: Every mathematical operation (addition, percentage, square root, squaring, sign inversion, and even retrieving the value of Pi) **MUST** be performed by calling the backend API. The frontend is strictly a thin UI layer.
2. **Huma v2 Marshalling**: Backend request/response structs must nest fields inside a `Body` struct to comply with Huma v2 standards and ensure correct JSON mapping.
3. **CORS Configuration**: The backend uses Chi middleware for CORS. Configure allowed origins via `CORS_ORIGINS` environment variable.
4. **Responsive UI**: The layout uses CSS variables combined with media queries for both `width` and `height`. It is optimized for iPhone devices, Android devices, and desktop monitors.
5. **Port Configuration**: Backend port is configurable via `PORT` environment variable (default: 8888).

## Features & Improvements

### Backend Features

- **Rate Limiting**: 5 requests/second per IP (configurable)
- **Security Headers**: X-Content-Type-Options, X-Frame-Options, X-XSS-Protection, Referrer-Policy, Permissions-Policy
- **HTTP Timeouts**: 15s read, 15s write, 60s idle
- **Graceful Shutdown**: Handles SIGINT/SIGTERM with 30s grace period
- **Structured Logging**: Using slog with JSON output
- **Environment Variables**: `PORT` and `CORS_ORIGINS`

### Frontend Features

- **Error Boundary**: Catches React render errors with user-friendly message
- **Loading States**: Visual spinner during API operations
- **Request Cancellation**: AbortController to cancel stale requests
- **Focus Trapping**: Modal accessibility with Tab/Shift+Tab cycling
- **Hidden Scrollbars**: Custom scrollbar styling
- **History Limit**: Max 50 entries in sessionStorage
- **Responsive Design**: Optimized for mobile and desktop

## Key Session History & Resolved Bugs

- **CORS Preflight**: Resolved 405 errors by using `cors.New().Handler` in the Go backend.
- **Huma Panic**: Fixed a panic caused by using the reserved field name `Status` (type string) in response structs; Huma expects `Status` to be an `int` for the HTTP code.
- **Display Clipping**: Implemented dynamic font scaling in the display to support 12+ digits.
- **iOS Responsiveness**: Centered the UI using full-screen Flexbox and prevented button overlap on small screens by scaling based on viewport height.
- **Percentage Context**: Fixed a logic bug where `100 - 8%` was calculating `8/100` instead of `8% of 100`.
- **Square/Percentage Overflow**: Added validation for Infinity/NaN results.

## Important Notes

- All mathematical operations MUST go through the backend API - no frontend arithmetic.
- Frontend uses `sessionStorage` for history (max 50 entries).
- Rate limiting is enforced per IP address.
